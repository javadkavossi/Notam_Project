# بررسی کارشناسی — سامانهٔ رتبه‌بندی اهمیت NOTAM

> **مخاطب:** کارشناس/کارشناسان هوانوردی (خلبان، دیسپچر، AIS).
> **هدف:** بازبینی صحت **دسته‌بندی** و **رتبه‌بندی اهمیت** NOTAMها پیش از استفادهٔ عملیاتی.
> **نسخهٔ منطق وزن‌دهی:** `1.0.0`

این سیستم به‌صورت خودکار هر NOTAM را می‌خواند، دسته‌بندی می‌کند و یک **امتیاز اهمیت (۰ تا ۱۰۰)** و یک **سطح** (بحرانی/بالا/متوسط/پایین/اطلاعی) به آن می‌دهد تا خلبان سریع‌تر موارد مهم را ببیند. ما به تأیید شما نیاز داریم که این قضاوت‌ها **از نظر عملیاتی درست‌اند**.

---

## چرا نظر شما حیاتی است

این یک سامانهٔ ایمنی‌محور است. دو خطای متضاد هر دو خطرناک‌اند:

- **کم‌گویی (Under-alerting):** یک NOTAM واقعاً مهم، کم‌اهمیت نشان داده شود و خلبان از کنارش رد شود.
- **گرگ‌گویی (Over-alerting / Cry Wolf):** آن‌قدر مورد «بحرانی» نشان داده شود که خلبان به کل هشدارها بی‌اعتماد شود.

منطق فعلی کاملاً **قاعده‌محور و قابل‌توضیح** است (بدون هوش مصنوعی/حدس) — یعنی هر امتیاز دقیقاً قابل ردیابی است و طبق نظر شما قابل تنظیم. ما وزن‌ها را بر اساس مهندسی گذاشته‌ایم؛ **قضاوت عملیاتی با شماست.**

---

## سیستم چطور امتیاز می‌دهد (خلاصهٔ غیرفنی)

هر NOTAM یک کد استاندارد ICAO پنج‌حرفی دارد (مثل `QMRLC`) که ماشین‌خوان است:

```
Q M R L C
  └┬┘ └┬┘
موضوع  وضعیت
 MR=باند   LC=بسته      →  «باند بسته»
```

امتیاز از سه بخش ساخته می‌شود:

```
امتیاز = وزنِ دستهٔ موضوع  +  تعدیلِ وضعیت  +  امتیازِ پرچم‌های بحرانی
```

- **وزن دسته:** باند از تاکسی‌وی مهم‌تر است، فرودگاه از هر دو مهم‌تر.
- **تعدیل وضعیت:** «بسته» امتیاز را بالا می‌برد، «لغو/تکمیل‌شده» پایین می‌آورد.
- **پرچم‌ها:** موارد بحرانی خاص (باند بسته، فرودگاه بسته، FICON، خرابی ILS/GPS).

سپس امتیاز به سطح نگاشت می‌شود:

| سطح | بازهٔ امتیاز | نمایش برای خلبان |
|-----|:---:|------|
| 🔴 **بحرانی (CRITICAL)** | ۸۰–۱۰۰ | بولد، بالای فهرست، اخطار اجباری |
| 🟠 **بالا (HIGH)** | ۶۰–۷۹ | برجسته |
| 🟡 **متوسط (MEDIUM)** | ۳۵–۵۹ | عادی |
| 🟢 **پایین (LOW)** | ۱۵–۳۴ | جمع‌شده |
| ⚪ **اطلاعی (INFO)** | ۰–۱۴ | جمع‌شده |

> **مهم:** یک NOTAM می‌تواند برای یک پرواز بحرانی و برای پرواز دیگر کم‌اهمیت باشد. مثلاً «باند ۲۹ بسته» اگر پرواز روی همان باند فرود بیاید بحرانی است، ولی اگر از باند دیگری استفاده کند کمتر. این «تعدیل بر اساس پرواز» جدا از امتیاز پایه انجام می‌شود؛ **در این بررسی روی امتیاز پایه تمرکز کنید.**

## ۱) جدول وزن دسته‌ها — لطفاً بازبینی کنید

آیا این ترتیب و مقادیر با اولویت عملیاتی واقعی همخوان است؟

| دسته | وزن پایه | توضیح | نظر شما |
|------|:---:|------|:---:|
| فرودگاه (AERODROME) | 78 | کل فرودگاه (فقط `FA`/`FP`) | |
| باند (RUNWAY) | 70 | باند، آستانه، فاصله‌های اعلامی | |
| آتش‌نشانی/نجات (RESCUE) | 62 | RFFS | |
| فضای هوایی (AIRSPACE) | 55 | ساختار، FIR، مسیر | |
| محدودیت (RESTRICTION) | 55 | ممنوعه/خطرناک/محدود | |
| خدمات ATS (ATS) | 55 | برج، نزدیکی، رادار، FIS | |
| ILS | 50 | سامانهٔ فرود دقیق | |
| رویه‌ها (PROCEDURE) | 50 | SID/STAR/نزدیکی | |
| GPS/GNSS | 50 | | |
| هواشناسی (MET) | 45 | RVR، باد، سقف، اصطکاک | |
| هشدار (WARNING) | 45 | تیراندازی، پهپاد، بالن | |
| مانع (OBSTACLE) | 45 | | |
| ناوبری (NAVIGATION) | 45 | VOR/DME/NDB | |
| روشنایی (LIGHTING) | 40 | روشنایی باند/نزدیکی | |
| ارتباطات (COMMS) | 40 | رادار/مخابرات | |
| تاکسی‌وی (TAXIWAY) | 38 | | |
| سطح حرکت — نامشخص (MOVEMENT_AREA) | 35 | کد گروه M که موضوع دقیقش شناخته نشده | |
| خدمات فرودگاهی (SERVICE) | 28 | سوخت، اکسیژن، گمرک، برف‌روبی | |
| سایر (OTHER) | 20 | اداری/متفرقه | |

**سؤالات کلیدی:**
- آیا `RESCUE` (نبود آتش‌نشانی) در جای درستی است؟ در بسیاری کشورها نبود RFFS = تعطیلی عملی فرودگاه برای پروازهای تجاری.
- آیا `MET` (نبود RVR/باد) باید بالاتر باشد؟ برای عملیات کم‌دید حیاتی است.
- آیا `SERVICE=28` (نبود سوخت) درست است؟ برای پرواز طولانی که آنجا سوخت‌گیری برنامه‌ریزی شده، ممکن است مهم‌تر باشد — ولی این «وابسته به پرواز» است.

## ۴) نمونه‌های واقعی — لطفاً «سطح درست» را تأیید یا اصلاح کنید

## ۵) توزیع کلی — آیا متعادل است؟

سیستم روی **۹۰۵ NOTAM واقعی** این توزیع را تولید کرد:

| سطح | تعداد | درصد |
|-----|:---:|:---:|
| 🔴 بحرانی | ۹۴ | ٪۱۰ |
| 🟠 بالا | ۴۴۳ | ٪۴۹ |
| 🟡 متوسط | ۲۷۸ | ٪۳۱ |
| 🟢 پایین | ۴۸ | ٪۵ |
| ⚪ اطلاعی | ۴۲ | ٪۵ |

**سؤال کلیدی:** آیا **٪۴۹ در سطح «بالا»** بیش از حد است؟ اگر تقریباً نیمی از NOTAMها «بالا» باشند، این سطح معنای خودش را از دست می‌دهد. آیا باید:
- (الف) آستانهٔ «بالا» را از ۶۰ به ۶۵ یا ۷۰ برد؟
- (ب) وزن پایهٔ برخی دسته‌ها را کم کرد؟
- (ج) همین درست است چون داده مربوط به دورهٔ پرتنش (بحران اوکراین، فوریه ۲۰۲۲) است؟

> **توجه:** غلبهٔ دستهٔ «محدودیت فضای هوایی» (۳۲۳ از ۹۰۵) طبیعی است — داده دقیقاً مربوط به زمان بستن گستردهٔ فضای هوایی اروپای شرقی است.

## ۶) پوشش Q-code — آیا موردی جا مانده؟

سیستم جدول کدهای ICAO را دارد، ولی همه را پوشش نداده. کدهای زیر در داده دیده شدند ولی **کامل شناسایی نشدند** (به دستهٔ کلی افتادند). لطفاً بگویید کدام‌ها مهم‌اند:

| کد ناقص | تعداد | حدس سیستم | معنی احتمالی |
|---------|:---:|-----------|------|
| `SP` (approach control) | ۱۷ | خدمات ATS | کنترل نزدیکی |
| `ST` (tower) | ۱۲ | خدمات ATS | برج مراقبت |
| `MA` (movement area) | ۴ | سطح حرکت | کل محوطهٔ مانور |
| `SE` (FIS) | ۳ | خدمات ATS | خدمات اطلاعات پرواز |
| `FU` (fuel) | ۴ | خدمات | سوخت |

(این‌ها الان با «حدس گروه» کار می‌کنند؛ اگر تأیید کنید معنی دقیقشان را، به جدول اضافه می‌کنیم.)

## چه چیزی باید در نظر بگیرید (چک‌لیست)

هنگام بررسی، لطفاً به این‌ها فکر کنید:

1. **آیا سطح با واکنش عملیاتی خلبان می‌خواند؟** «بحرانی» یعنی «ممکن است پرواز را لغو/منحرف کنی». «بالا» یعنی «حتماً بخوان و در تصمیم لحاظ کن». آیا نمونه‌های بخش ۴ با این تعریف جور است؟
2. **گرگ‌گویی:** آیا موردی هست که به‌غلط بحرانی/بالا شده و باعث بی‌اعتمادی می‌شود؟
3. **کم‌گویی:** آیا موردی هست که سیستم دست‌کم گرفته و باید بالاتر باشد؟
4. **تفاوت فاز پرواز:** آیا دسته‌ای هست که برای فاز خاصی (مثلاً فرود در دید کم) اهمیت متفاوتی دارد که سیستم لحاظ نکرده؟
5. **موارد خاص کشوری/منطقه‌ای:** آیا رویه‌ای هست که در منطقهٔ عملیاتی شما معنای متفاوتی دارد؟
6. **پرچم‌های جامانده:** آیا نوع NOTAMی هست که باید همیشه برجسته شود ولی در بخش ۳ نیست؟

---

## چطور بازخورد بدهید

هر شکلی که راحت‌تر است:
- روی همین جدول‌ها (بخش ۱ تا ۴) مستقیم بنویسید یا ستون آخر را پر کنید.
- یا فهرست ساده: «Q-code فلان باید سطح فلان باشد، چون...».
- به‌ویژه روی **موارد اختلافی** و **موارد جامانده** تمرکز کنید — همان‌ها بیشترین ارزش را دارند.

هر تغییری که تأیید کنید، در جدول وزن‌ها اعمال و نسخهٔ منطق (`1.0.0`) افزایش داده می‌شود و **همهٔ NOTAMهای موجود با منطق جدید بازپردازش می‌شوند** (بدون نیاز به دریافت مجدد).
