# گزارش کالیبراسیون امتیازدهی — نسخهٔ 1.2.0

> **هشدار:** این کالیبراسیون بر پایهٔ بازخورد اولیهٔ کارشناس است و **هنوز نهایی نیست**. هیچ ادعایی دربارهٔ دقیق‌بودن قطعیِ وزن‌ها پیش از تأیید نهایی کارشناس نمی‌شود. اعداد «سقف بالقوه»اند، نه امتیاز نهایی.

نسخهٔ فعال: **1.2.0** (`EffectiveFrom 2026-07-19`) · baseline حفظ‌شده: **1.1.0**
تولیدشده از دادهٔ واقعی: **۹۴۳ NOTAM** (فوریه ۲۰۲۲، FAA).

---

## ۱. اصل کلیدی: سقف بالقوه ≠ امتیاز نهایی

```
Evidence → Base Potential → Operational Impact → Flight Context → Final Score / Priority / Action
```

- **وزن دسته = سقف بالقوه** (بدترین شرایط ممکن). لایهٔ Operational Impact آن را کاهش می‌دهد، `NOT_APPLICABLE` می‌کند، یا در صورت اثبات اثر واقعی حفظ می‌کند.
- هر نتیجه `scoringVersion` دارد → قابل بازتولید. نسخهٔ 1.1.0 به‌عنوان baseline در `registry` حفظ شده.
- کدِ نسخه‌بندی: [`internal/pipeline/analysis/scoring_config.go`](../../src/internal/pipeline/analysis/scoring_config.go).

| سطح | سهم نهایی |
|-----|:---:|
| 🔴 CRITICAL | ۵ (۳٪) |
| 🟠 **HIGH** | ۱۰ (**۶٪**) |

توزیع `contextResult` فضای هوایی (تجمیع سناریوها): `NO_INTERSECTION 629` · `FULL_INTERSECTION 32` · `HORIZONTAL_ONLY 3` · `UNKNOWN_FLIGHT_LEVEL 1` — یعنی اکثر محدودیت‌های فضای هوایی که با FIR match شده‌اند، با مسیر/ارتفاع واقعی تداخل ندارند و درست de-escalate می‌شوند.

گزارش کامل با `go run ./cmd/calibration` بازتولید می‌شود. ۳۰ نمونهٔ مرزی در [`calibration_samples.json`](calibration_samples.json).

## ۵. موارد اختلافی/مرزی که باید کارشناس ببیند

- **MET = 70:** با هر U/S (مثل سنسور باد) به ۹۲/CRITICAL پایه می‌رسد. آیا این «سقف بالقوه» درست است؟ (در corpus حکم نهایی نمی‌گیرد؛ `CONTEXT_REQUIRED`.)
- **ILS U/S = 83 (CRITICAL پایه):** با سقف ۵۵ + U/S 22 + پرچم 6. آیا در حد بالقوه درست است؟
- ۳۰ نمونهٔ [`calibration_samples.json`](calibration_samples.json): ۵ افزایش پایه، ۱۰ کاهش کانتکستی، ۵ مرز MEDIUM/HIGH، ۵ مرز LOW/MEDIUM، ۵ اطمینان‌پایین.

## ۶. محافظ‌های ایمنی (تست‌شده)

- Base Potential بالا به‌تنهایی **حکم نهاییِ CRITICAL نمی‌دهد** (corpus → `CONTEXT_REQUIRED`). `TestCorpusNeverFinalCritical`
- `NOT_APPLICABLE` → امتیاز نزدیک صفر (سقف `NotApplicableCap=20`).
- دادهٔ نامعلوم → **هرگز** به خطرناک‌ترین حالت fallback نمی‌شود (سقف `UnknownCap=59`). `TestAirspace_UnknownsNeverEscalate`
- Low Confidence → حکم قطعیِ «فرودگاه غیرقابل‌استفاده» نمی‌دهد.
- CRITICAL نهایی فقط با context تأییدشده (تک‌باند / FULL_INTERSECTION). `TestCriticalRequiresConfirmedContext`
- Trigger/Informational به‌خاطر وزن دسته HIGH نمی‌شوند.
- تغییر نسخه → نتیجهٔ قبلی قابل بازتولید. `TestScoringVersionReproducibility`

## ۸. جمع‌بندی برای کارشناس

۱. سقف‌های پیشنهادی شما اعمال شد (نسخهٔ 1.2.0).
